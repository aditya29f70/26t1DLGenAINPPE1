## 1. Introduction

**Problem** : Chest X-ray imaging is one of the most widely used diagnostic tools in medical practice for detecting diseases related to the thoracic region such as lung infections, fluid accumulation, or structural abnormalities of the heart and lungs.

**Important** : Manual analysis of X-ray images by radiologists can be time-consuming and prone to variability due to human factors.

**Dataset** : Chest X-ray images labeled with thoracic pathologies.

- Number of Classes: There are 20 classes
  Training Dataset

- Training Dataset
  | Number of images 51,043 | Each image has a corresponding label indicating the presence of one pathology.

- Test Dataset | Number of images 17,015 | Labels not available

- Image belongs to exactly one class.

**Objective** : Automated deep learning system capable of detecting thoracic diseases from chest X-ray images.

## 2. Methodology

**Models Used**

1. Own designed CNN

- Two convolutional layers with maxpooling and in fully connected layer we had two dense layers (first with 128 nodes and second 84 nodes with **Relu** activation function) and one output layer (20 nodes with softmax activation function.)
- Weighted F1 score -4.79

2. Transfer Learning or Pretrained Models on **ImageNet** dataset.

- VGG16 (Why I used?)
- - VGG16 has simple and uniform architecture -> easy to fine-tune

- - Took convolution layers from **block5_conv1** and new fully connected layers in pretraining.
- - Weighted F1 score -4.62

- DenseNet121 (Why I used?)
- - dense connections preserve fine-grained texture features (nodules, infiltrates) that skip connections in ResNet lose. Lightweight enough to train fast. Proven on exact same pathology classes we have.

- - Took convolution layers from **conv5_block15_2_conv** and new fully connected layers in pretraining.

- - Weighted F1 score -4.72

**Preprocessing steps**

- Build Data Generator to prevent load all inputs at a time in memory or for fast training
- Batch size (32) -> For reducing load and fast training
- resizing to (224, 224)
- Dropout (p=0.2) -> For prevent overfitting
- Input Normalization -> Don't want oscillation or loss get stopped during training

**Techniques**

- Data Augmentations
- Trnasfer Learning
- Build a evaluation function **asymmetric_cost_score** which gives multi-class classification report (N_c, FP_c, FN_c, TP_c, score)

## 3. Experiments

**Experiments Performed**

- Changing Number of Trainable Parameters
- Tried to used different loss function like **Categorical cross-entropy**, **weighted_bce**, **focal_loss**, **asymmetric_loss**

## 6. Conclusion

1. Initial attempt: VGG16 (lightweight pretrained model)

- Didn't work well due to **heavily imbalanced classes** (**No disease** was in high numbers, So all the time models were giving more importance to this class)

2. Custom CNN architecture:

- 2 convolutional layers with max pooling
- Fully connected layers: 128 nodes -> 84 nodes (ReLU activation)
- Output layer: 20 nodes (softmax for multi-class)
- Better performance but still misclassifying underrepresented classes

3. Next steps: Moved to DenseNet and explored different loss functions to better handle class imbalance
